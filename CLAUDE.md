# Wiki Schema — Personal Second Brain

Это локальная база знаний, поддерживаемая LLM по паттерну Андрея Карпатого.
Архитектура: **raw/ → синтез → wiki/** — агент читает источники и компилирует знания в структурированные страницы.

---

## Язык

**Все wiki-страницы пишутся на русском языке — без исключений.**

Источники в `raw/` могут быть на любом языке (английский, китайский, etc.). Задача агента — перевести, осмыслить и синтезировать знания на русском. Не переводи дословно — извлекай суть и записывай в своих словах на русском.

Исключения (оставить в оригинале):
- Имена собственные, названия инструментов/библиотек/компаний (например: Obsidian, PyTorch, Anthropic)
- Устоявшиеся термины без адекватного перевода (например: backlink, embedding, fine-tuning) — писать в оригинале, при первом появлении добавить пояснение в скобках
- Цитаты из источника — оформлять как цитаты (`>`) и указывать язык оригинала

---

## Роль агента

- Агент **владеет** папкой `wiki/` — создаёт, обновляет и связывает страницы
- Агент **никогда не изменяет и не удаляет** файлы в `raw/` — они иммутабельны
- Агент **обновляет** `wiki/index.md` и `wiki/log.md` после каждой операции

---

## Структура папок

```
raw/articles/   — статьи, web clips, блог-посты
raw/books/      — книги, конспекты, заметки по книгам
raw/misc/       — всё остальное, ручные заметки
wiki/           — синтезированные страницы знаний (Obsidian vault)
wiki/index.md   — каталог всех wiki-страниц
wiki/log.md     — аудит-журнал всех операций
```

---

## Форматирование wiki-страниц

Каждая страница в `wiki/` — Obsidian-совместимый markdown с обязательным frontmatter:

```markdown
---
title: "Название страницы"
tags: [тег1, тег2]
created: YYYY-MM-DD
updated: YYYY-MM-DD
sources:
  - raw/articles/YYYY-MM-DD-slug.md
---

# Название

Тело страницы...

## See Also
- [[связанная-страница]]
```

**Обязательные поля frontmatter:**
- `title` — полное название страницы
- `tags` — массив тегов (используй существующие теги из других страниц, если подходят)
- `created` — дата создания (YYYY-MM-DD)
- `updated` — дата последнего обновления (YYYY-MM-DD), обновлять при каждом изменении
- `sources` — список исходных файлов в raw/, которые использовались для создания страницы

---

## Правила ссылок

- **Все ссылки между wiki-страницами — только `[[wikilink]]` формат** (Obsidian граф работает только с ними)
- Имя в `[[wikilink]]` = имя файла без `.md` (например, `[[machine-learning]]` для `wiki/machine-learning.md`)
- Ссылки на raw/ файлы в теле страницы: стандартный markdown `[название](../raw/articles/slug.md)`
- Секция `## See Also` обязательна если есть хотя бы одна связанная страница

---

## Именование файлов

- Формат: `lowercase-hyphenated-slug.md`
- Только латиница, цифры и дефисы
- Максимум 50 символов
- Примеры: `machine-learning.md`, `atomic-habits.md`, `zettelkasten-method.md`

---

## Правила backlink-аудита (КРИТИЧНО)

При создании или существенном обновлении страницы — **обязательно**:
1. Прочитать `wiki/index.md` для списка всех существующих страниц
2. Для каждой связанной существующей страницы — добавить `[[новый-slug]]` в её секцию `## See Also`
3. Зафиксировать все добавленные backlinks в записи `wiki/log.md`

Без этого шага wiki превращается в несвязанные острова.

---

## wiki/index.md — формат

```markdown
# Wiki Index

| Страница | Теги | Резюме | Обновлено |
|----------|------|--------|-----------|
| [[slug]] | tag1, tag2 | Одно предложение резюме | YYYY-MM-DD |
```

- Каждая wiki-страница = одна строка в таблице
- Резюме — максимум одно предложение
- Сортировка: по алфавиту внутри тематических групп (необязательно жёсткая)

---

## wiki/log.md — формат

```markdown
# Wiki Log

## YYYY-MM-DD HH:MM — ОПЕРАЦИЯ
- Источник: `путь/к/файлу`
- Создано: [[page1]], [[page2]]
- Обновлено: [[page3]]
- Backlinks добавлены в: [[page4]], [[page5]]
```

- Только дописывать новые записи — **никогда не редактировать прошлые**
- При превышении 500 строк — архивировать в `log-YYYY.md`, начать новый log.md с заголовком `# Wiki Log`

---

## Deduplication

Перед созданием новой страницы — проверить `wiki/index.md`:
- Если страница с похожей темой уже существует — обновить её, а не создавать новую
- Создавать новую страницу только если тема достаточно самостоятельна

---

## Теги — рекомендуемые

Используй теги из уже существующих страниц. Базовые категории для личного second brain:
- `books`, `articles`, `ideas`, `people`, `tools`
- `ml`, `programming`, `productivity`, `science`
- `query-answer` — для страниц, сохранённых из wiki-query
